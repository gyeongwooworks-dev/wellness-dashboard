# PRD — 웰니스 웨어러블 사용자 인사이트 대시보드

마케팅팀·제품팀이 함께 참고하는 사내 분석 대시보드의 제품 요구사항 정의서.

| 항목 | 내용 |
|---|---|
| 작성자 | 제품 데이터 분석가 |
| 대상 팀 | 마케팅팀(1차), 제품팀(1차) |
| 문서 버전 | v1.0 (Draft) |
| 데이터 | 웨어러블 디바이스 활동·수면·심박·칼로리 로그 (샘플: FitBit Fitness Tracker Data, 2개 기간 병합) |
| 대시보드 형태 | 공통 KPI 바 + 팀별 섹션 (마케팅 / 제품) |

---

## 1. 배경 및 목적

우리 회사는 웰니스 웨어러블 기기를 만들고, 기기는 걸음·활동·칼로리·수면·심박 데이터를 지속적으로 수집한다. 현재 이 데이터는 분석가가 임시 쿼리로 개별 확인할 뿐, **마케팅팀과 제품팀이 스스로 참고할 수 있는 정리된 뷰가 없다.**

이 대시보드의 목적은 두 팀이 각자의 의사결정에 필요한 사용자 행동 인사이트를 **한 곳에서, 동일한 정의로** 볼 수 있게 하는 것이다.

- **마케팅팀**: 활동·수면 패턴으로 사용자를 세분화하고 페르소나를 정의해, 타깃 캠페인·메시지·콘텐츠 소재로 활용.
- **제품팀**: 활동·수면·심박 간 관계에서 기능별 행동 인사이트를 얻어, 로드맵·기능 우선순위 판단에 활용.

### 핵심 질문 (이 대시보드가 답해야 할 것)
- 우리 사용자는 어떤 행동 유형으로 나뉘는가? 각 세그먼트는 얼마나 큰가?
- 사용자는 시간이 지나도 꾸준히 기기를 쓰는가(참여 지속성)?
- 수면과 활동은 어떤 관계가 있고, 제품이 개입할 여지는 어디인가?
- 언제(시간대·요일) 활동이 집중되어 알림·기능 노출 타이밍을 잡을 수 있는가?

## 2. 목표 및 성공 지표

### 2.1 비즈니스 목표
| 팀 | 목표 | 이 대시보드의 기여 |
|---|---|---|
| 마케팅 | 세그먼트 기반 타깃팅으로 캠페인 효율 향상 | 세그먼트 규모·특성·페르소나 카드 제공 |
| 제품 | 데이터 근거 기반 기능 우선순위 결정 | 행동 관계·참여 지속성 지표 제공 |
| 공통 | 지표 정의 단일화(single source of truth) | 공통 KPI 바 + 표준 지표 사전 |

### 2.2 대시보드 성공 지표 (KPI of the dashboard)
- 두 팀이 월 1회 이상 실제 의사결정 회의에서 인용.
- 세그먼트 정의가 마케팅 캠페인 1건 이상, 제품 백로그 1건 이상에 반영.
- 지표 정의 관련 문의(“이 숫자 어떻게 나온 거냐”)가 도입 후 감소.

## 3. 데이터 소스 및 정의

### 3.1 데이터 개요 (현재 샘플)
| 데이터 | 단위 | 커버리지 | 기간 |
|---|---|---|---|
| 활동/걸음/칼로리 (daily) | 1일 | 사용자 35명 · 62일 | 2016-03-12 ~ 05-12 (2개 기간 병합) |
| 시간대별 활동/칼로리/강도 | 1시간 | 전체 | 전 기간 |
| 심박수 | 초 → 분 집계 | 15명 | 전 기간 |
| 수면 (일별 요약) | 1일 | 24명 | 2016-04-12 ~ 05-12 |

> **주의**: 현재는 공개 샘플 데이터(2016·소표본)를 사용한 프로토타입이다. 프로덕션에서는 동일 스키마를 자사 기기 텔레메트리에 연결한다. 커버리지가 데이터별로 다르므로(심박 15명·수면 24명) 각 지표에 **표본 수를 함께 표기**한다.

### 3.2 표준 지표 정의 (지표 사전)
| 지표 | 정의 |
|---|---|
| 활성 시간 | `VeryActiveMinutes + FairlyActiveMinutes` |
| 활동 강도 분포 | Very/Fairly/Lightly/Sedentary 분(minute) 비율 |
| 수면 효율(%) | `총 수면시간 ÷ 침대에 있던 시간 × 100` |
| 기록 지속성(참여 대리지표) | 관측기간 중 데이터가 기록된 일수 ÷ 전체 가능일수 |
| 심박 존 | 휴식 `<100` / 지방연소 `100–120` / 유산소 `120–140` / 고강도 `≥140` bpm |

> 수면 단계(깊은/얕은 잠) 원천 데이터가 없으므로, 수면의 질은 **수면 효율**로 대체 정의한다.

## 4. 사용자 세분화 방법론 (마케팅 핵심)

세그먼트는 **활동량 × 참여 지속성 × 수면**을 축으로 규칙 기반으로 정의한다(초기 버전). 향후 군집화(k-means 등)로 고도화 가능.

### 4.1 활동량 세그먼트
| 세그먼트 | 기준(예시, 조정 가능) |
|---|---|
| 고활동 (Power Mover) | 일 평균 걸음 ≥ 10,000 또는 활성 시간 상위 25% |
| 중활동 (Steady) | 일 평균 걸음 5,000–10,000 |
| 저활동 (Sedentary-leaning) | 일 평균 걸음 < 5,000 |

### 4.2 참여 세그먼트
| 세그먼트 | 기준 |
|---|---|
| 코어 유저 | 기록 지속성 ≥ 80% |
| 캐주얼 유저 | 40–80% |
| 이탈 위험 | < 40% 또는 최근 기록 급감 |

### 4.3 페르소나 카드(마케팅 산출물)
각 세그먼트를 대표 지표(평균 걸음·칼로리·수면효율·기록 지속성)와 한 줄 설명, 추천 메시지 방향으로 카드화한다. 예: “**아침형 파워무버** — 오전 활동 집중, 코어 유저. 챌린지·목표 기능 소구.”

## 5. 대시보드 구조

스크롤형 단일 페이지. 상단 공통 KPI 바 아래에 팀별 섹션을 탭 또는 앵커로 구분.

```
[헤더 + 관측기간·표본 표기]
[전역 필터: 기간 / 세그먼트 / 사용자]
[공통 KPI 바]  ← 두 팀 공유
────────────────────────
[A. 마케팅 섹션]  세분화 · 페르소나
[B. 제품 섹션]    기능별 행동 인사이트 · 참여 지속성
────────────────────────
[지표 사전 · 데이터 유의사항 푸터]
```

### 5.1 공통 KPI 바
평균 걸음 · 평균 칼로리 소모 · 평균 활성 시간 · 평균 수면 효율 · 평균 안정 심박 · 활성 사용자 수(기록 지속성 기준). 필터 변경 시 실시간 갱신.

## 6. 기능 요구사항

### 6.1 전역 필터 (P0)
- 기간 구간, 세그먼트(활동/참여), 개별 사용자 선택.
- 필터 변경 시 하위 모든 시각화 실시간 반영.
- 데이터 커버리지가 다른 지표는 대상 표본 수를 항상 노출.

### A. 마케팅 섹션 — 사용자 세분화 / 페르소나

| ID | 기능 | 시각화 | 우선순위 |
|---|---|---|---|
| M1 | 세그먼트 규모 분포 | 도넛/막대 (활동·참여 세그먼트별 사용자 수·비중) | P0 |
| M2 | 세그먼트별 특성 비교 | 레이더/그룹 막대 (걸음·칼로리·수면효율·지속성) | P0 |
| M3 | 페르소나 카드 | 세그먼트별 대표 지표 + 메시지 방향 카드 | P1 |
| M4 | 활동 집중 시간대 | 시간대별 히트맵/막대 (캠페인·알림 타이밍) | P1 |

### B. 제품 섹션 — 기능별 행동 인사이트 / 참여 지속성

| ID | 기능 | 시각화 | 우선순위 |
|---|---|---|---|
| P1 | 활동·칼로리 추이/패턴 | 일별 라인·영역 + 시간대별 강도 | P0 |
| P2 | 수면–다음날 활동 관계 | 산점도 + 추세선 + 상관계수 | P0 |
| P3 | 기록 지속성(리텐션 대리) | 기간별 기록 유지율 / 코호트 잔존 곡선 | P0 |
| P4 | 심박 기반 운동강도 | 세션 심박 시계열 + 존 비중 | P1 |
| P5 | 데이터 품질/센서 커버리지 | 지표별 기록 사용자 수·결측 현황 | P2 |

각 시각화 하단에는 **해석 문구 1줄**(예: “수면 효율과 다음날 걸음의 상관은 약함 → 수면 개선만으로 활동 증가를 기대하기 어려움”)을 포함해, 비분석 직군도 바로 이해하도록 한다.

---

## 7. 비기능 요구사항
- **정의 일관성**: 모든 지표는 3.2 지표 사전을 단일 출처로 사용.
- **성능**: 초기 렌더 3초 이내(심박 등 대용량은 사전 집계).
- **접근성**: 색맹 고려 팔레트, 차트마다 단위·범례·표본 수 명시.
- **신뢰성**: 표본이 작은 지표(심박·수면)는 시각적으로 “참고” 표기.
- **갱신 주기**: 프로덕션 연결 시 일 1회 배치 갱신을 전제.

---

## 8. 범위

### In Scope
- 공통 KPI 바 + 마케팅·제품 팀별 섹션.
- 규칙 기반 세그먼트/페르소나, 4개 분석 테마(활동·칼로리, 수면-활동, 세분화, 참여 지속성).

### Out of Scope (이번 버전 제외)
- 실시간 스트리밍, 개인 식별(PII) 연계, 푸시 알림 자동화.
- 머신러닝 군집화·예측(향후 확장).
- 수면 단계(깊은/얕은 잠) 분석 — 원천 데이터 없음.

## 9. 리스크 및 데이터 유의사항
| 구분 | 내용 | 대응 |
|---|---|---|
| 표본 대표성 | 현재 샘플 35명·2016년 | 프로토타입임을 명시, 프로덕션 데이터로 재검증 |
| 커버리지 불균형 | 심박 15명·수면 24명 | 지표별 표본 수 병기, 소표본 “참고” 표기 |
| 세그먼트 임의성 | 규칙 기준 컷오프 | 기준을 설정값으로 노출, 팀 합의 후 확정 |
| 인과 오해 | 상관을 인과로 해석 위험 | 해석 문구에 “상관 ≠ 인과” 명시 |

## 10. 향후 확장
- 군집화 기반 자동 세그먼트 및 페르소나 정교화.
- 코호트 리텐션 정식 분석(가입 시점 기준 잔존율).
- 마케팅 캠페인 성과와 세그먼트 연결(캠페인 → 행동 변화 측정).
- 자사 실기기 텔레메트리 연결 및 일 배치 파이프라인.
